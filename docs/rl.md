## 強化学習の基礎

### 教師あり学習との違い

教師あり学習では「正解ラベル」が与えられる。強化学習では「報酬」しか得られない。

- 正解ラベル: 「この入力に対する正しい出力はこれ」（直接的な指導）
- 報酬: 「さっきの行動は良かった/悪かった」（評価的なフィードバック）

エージェントは試行錯誤を通じて、累積報酬を最大化する方策（行動の選び方）を学習する。

### マルコフ決定過程 (MDP)

強化学習は通常MDP (S, A, P, R, γ) としてモデル化される：

- **S**: 状態の集合
- **A**: 行動の集合
- **P(s'|s,a)**: 遷移確率（状態sで行動aを取ったときに状態s'に遷移する確率）
- **R(s,a,s')**: 報酬関数
- **γ**: 割引率 (0 ≤ γ ≤ 1)

マルコフ性: 次の状態は現在の状態と行動にのみ依存し、過去の履歴には依存しない。

### 価値関数

#### 状態価値関数 V(s)

状態sからスタートして方策πに従ったときの期待累積報酬：

```
V^π(s) = E_π[Σ γ^t r_t | s_0 = s]
```

#### 行動価値関数 Q(s,a)

状態sで行動aを取り、その後方策πに従ったときの期待累積報酬：

```
Q^π(s,a) = E_π[Σ γ^t r_t | s_0 = s, a_0 = a]
```

Q関数はV関数より情報量が多い。方策を明示的に持たなくても、Q(s,a)が最大となるaを選べばよい。

### ベルマン方程式

価値関数の再帰的な定義。強化学習の理論的基盤：

```
V(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a) [R(s,a,s') + γ V(s')]
Q(s,a) = Σ_{s'} P(s'|s,a) [R(s,a,s') + γ Σ_{a'} π(a'|s') Q(s',a')]
```

「現在の価値 = 即時報酬 + 割引された将来の価値」

### 探索と活用のジレンマ

強化学習の本質的な困難：
- **活用 (Exploitation)**: 現時点で最良と思われる行動を選ぶ
- **探索 (Exploration)**: 未知の行動を試してより良い方策を発見する

活用ばかりでは局所最適に陥り、探索ばかりでは得た知識を活かせない。

## 多腕バンディット問題

### 概念

K本のスロットマシン（腕）があり、各腕の報酬分布は未知。報酬を最大化するためにどの腕をどの順番で引くか？

MDPの最も単純な特殊ケース（状態が1つだけ、遷移なし）。探索と活用のジレンマを最も純粋な形で体現する問題。

### Epsilon-Greedy

```
確率ε: ランダムに腕を選択（探索）
確率1-ε: 推定価値が最高の腕を選択（活用）
```

行動価値の推定はインクリメンタル平均で更新：

```
Q(a) ← Q(a) + (1/N(a)) * (r - Q(a))
```

この更新式は `新しい推定 = 古い推定 + ステップサイズ * (実測値 - 古い推定)` という構造。ステップサイズ `1/N(a)` は選択回数が増えるほど小さくなり、推定が安定する。

#### εの選び方

- ε=0: 純粋な活用。初期の推定が間違っていたら永遠に間違い続ける
- ε=1: 純粋な探索。情報は集まるが活かさない
- 実用的: ε=0.1 が一般的。εを時間とともに減衰させる方法もある

### UCB (Upper Confidence Bound)

```
UCB(a) = Q(a) + c * √(ln(t) / N(a))
```

εグリーディが「探索をランダムに行う」のに対し、UCBは「不確実性が高い腕を優先的に探索する」。

第2項は**不確実性ボーナス**：
- `N(a)` が小さい（あまり選ばれていない）腕はボーナスが大きい → 探索される
- `t` が大きくなるほどボーナスが増加 → 長期的にはすべての腕が十分に探索される
- `c` は探索の度合いを制御するハイパーパラメータ

UCBの理論的な強みは**後悔 (regret)** の上界が対数的であることが証明されている点。つまり最適な腕との報酬の差は O(log t) のペースでしか増えない。

## 動的計画法

### 前提

環境モデル（遷移確率P、報酬関数R）が**完全に既知**である場合に使えるアルゴリズム。

実世界ではモデルが既知であることは稀だが、強化学習の理論的基盤を理解するうえで重要。

### 価値反復法 (Value Iteration)

ベルマン最適方程式を反復的に適用して最適価値関数を求める：

```
V(s) ← max_a Σ_{s'} P(s'|s,a) [R(s,a,s') + γ V(s')]
```

全状態について上式で更新を繰り返し、変化量が閾値θ以下になったら収束とみなす。

収束後、最適方策は：

```
π*(s) = argmax_a Σ_{s'} P(s'|s,a) [R(s,a,s') + γ V(s')]
```

#### 収束性

縮小写像の定理により、γ < 1 であれば一意の不動点（最適価値関数）に収束することが保証される。

### 方策反復法 (Policy Iteration)

価値反復が「最適価値関数を直接求める」のに対し、方策反復は「方策の評価と改善を交互に行う」。

1. **方策評価**: 現在の方策πのもとで価値関数を計算
   ```
   V^π(s) = Σ_{s'} P(s'|s,π(s)) [R + γ V^π(s')]
   （収束するまで反復）
   ```

2. **方策改善**: 価値関数に基づき方策を更新
   ```
   π'(s) = argmax_a Σ_{s'} P(s'|s,a) [R + γ V^π(s')]
   ```

3. 方策が変化しなくなるまで1-2を繰り返す

### 2つの手法の比較

- 価値反復: 各反復で全状態のmaxを取る。反復回数は多いが1回の計算が軽い
- 方策反復: 方策評価で連立方程式を解く必要がある。反復回数は少ないが1回の計算が重い

実用上は方策反復の方が少ない反復で収束することが多い。

### 本実装のGridWorld

4×4のグリッドで状態0（左上）と状態15（右下）がゴール。各移動で-1の報酬。壁にぶつかるとその場に留まる。

gamma=1.0（割引なし）の場合、壁に向かって動き続ける方策だと V(s) = -1 + V(s) となり発散する。そのため方策反復ではデフォルト gamma=0.99 としている。

## TD学習

### 概念

TD (Temporal Difference) 学習は、動的計画法（モデルが必要）とモンテカルロ法（エピソード完了が必要）の中間に位置する。**モデル不要**で、**エピソード途中でも学習**できる。

### TD更新の核心

```
V(s) ← V(s) + α [r + γ V(s') - V(s)]
              ↑     ↑___TD目標___↑
              |         ↑
         ステップサイズ  TD誤差
```

TD誤差 `δ = r + γV(s') - V(s)` は「実際に得た報酬 + 次状態の推定価値」と「現在の推定価値」の差。この差を少しずつ縮めていく。

モンテカルロ法が実際の累積報酬（エピソード終了後に確定）で更新するのに対し、TD法は推定値で推定値を更新する（ブートストラップ）。

### Q-Learning (Off-policy)

```
Q(s,a) ← Q(s,a) + α [r + γ max_{a'} Q(s',a') - Q(s,a)]
```

- **Off-policy**: 行動方策（εグリーディ）と学習方策（greedy）が異なる
- TD目標に `max` を使うため、実際に取った行動に関係なく最適なQ値を学習する
- 探索をしながらも最適方策を直接学習できる

#### Off-policyの意味

Q-Learningは「実際にはεの確率でランダムに動く」が、「次の状態では最適な行動を取ると仮定」して学習する。現在の行動方策に縛られず、最適方策を目指す。

### SARSA (On-policy)

```
Q(s,a) ← Q(s,a) + α [r + γ Q(s',a') - Q(s,a)]
```

- **On-policy**: 実際に取った行動a'を使って更新
- TD目標に実際の次の行動a'を使うため、探索行動の影響を受ける

名前の由来: (S, A, R, S', A') の五つ組を使うから。

### Q-Learning vs SARSA：Cliff Walking での違い

崖歩きの環境で違いが顕著に現れる：

- **Q-Learning**: 崖の際を最短経路で歩く方策を学習。最適だが、εの探索で崖に落ちるリスクがある
- **SARSA**: 崖から離れた安全な経路を学習。最適ではないが、探索中の安全性を考慮する

Q-Learningは「理想上の最適方策」を学び、SARSAは「探索込みで安全な方策」を学ぶ。

## REINFORCE

### 概念

方策を直接パラメータ化し、期待報酬を最大化するように勾配上昇法で学習する。

価値ベース手法（Q-Learning, DQN）が行動価値を学習し、そこから間接的に方策を導出するのに対し、方策勾配法は方策そのものを直接最適化する。

### 方策勾配定理

```
∇J(θ) = E_π[Σ_t ∇log π(a_t|s_t; θ) * G_t]
```

- `π(a|s; θ)`: パラメータθで定義された方策（本実装ではsoftmax線形方策）
- `G_t = Σ_{k=t}^T γ^{k-t} r_k`: 時刻tからの割引累積報酬
- `∇log π`: スコア関数

直感的解釈：
- `G_t > 0` の行動 → `log π(a|s)` を大きくする → その行動が選ばれやすくなる
- `G_t < 0` の行動 → `log π(a|s)` を小さくする → その行動が選ばれにくくなる

報酬が高かった行動の確率を上げ、低かった行動の確率を下げる。

### ベースライン

```
∇J(θ) = E_π[Σ_t ∇log π(a_t|s_t) * (G_t - b)]
```

期待値を変えずに分散を削減するテクニック。`b` は任意の値で良いが、`b = E[G_t]` （報酬の平均）が最も分散を小さくする。

本実装では各エピソードの `G_t` の平均をベースラインとして使用。

### なぜ方策勾配法が必要か

1. **連続行動空間**: Q-Learningは有限個の行動を前提とする。連続的な行動（ロボットの関節角度など）には方策勾配法が自然
2. **確率的方策**: 価値ベース手法はdeterministicな方策を導出するが、一部の問題（じゃんけんなど）では確率的方策が最適
3. **方策の形を直接指定できる**: ドメイン知識を方策の構造に組み込める

### REINFORCEの限界

- **高分散**: モンテカルロ的に1エピソード分のサンプルで勾配を推定するため分散が大きい
- **サンプル効率が悪い**: 各エピソードの経験を1回しか使わない
- **学習が不安定**: 分散の大きさから学習率の調整が難しい

これらの問題を軽減するために、Actor-Critic（方策勾配 + 価値関数の組み合わせ）が提案された。

## DQN

### 概念

DQN (Deep Q-Network) は Q-Learning を深層ニューラルネットワークで近似する手法。テーブル型Q-Learning（状態数×行動数のテーブル）では扱えない大きな/連続的な状態空間に対応する。

2015年にDeepMindがAtariゲームで人間を超える性能を示し、深層強化学習の始まりとなった。

### 2つの主要な工夫

テーブルQ-Learningを単純にニューラルネットに置き換えると学習が不安定になる。DQNはこの不安定性を解決する2つの工夫を導入した。

#### 1. Experience Replay

経験（s, a, r, s', done）をバッファに蓄積し、ランダムにサンプリングして学習する。

なぜ必要か：
- **データの相関を破壊**: 連続した経験は強く相関している（同じ場所で似た経験をする）。相関のあるデータで学習すると、最近の経験に偏って過去の知識を忘れる。ランダムサンプリングにより独立同分布 (i.i.d.) に近づける。
- **データの再利用**: 各経験を複数回学習に使える。サンプル効率が向上。

```python
class ReplayBuffer:
    def push(self, state, action, reward, next_state, done):
        self.buffer.append((state, action, reward, next_state, done))

    def sample(self, batch_size):
        # ランダムにbatch_size個の経験を取り出す
```

#### 2. Target Network

Q値の更新目標を計算する専用のネットワーク。一定間隔でQ-Networkのパラメータをコピーする。

なぜ必要か：
```
target = r + γ * max Q_target(s', a')
loss = (Q(s,a) - target)²
```

もし `Q` と `Q_target` が同じネットワークなら、Q(s,a) を更新すると Q(s',a') も変化し、目標自体が動いてしまう。「動く目標を追いかける」ことで学習が発振・発散する。

Target Networkを固定することで目標が安定し、教師あり学習に近い安定した最適化が可能になる。

### 学習アルゴリズム

```
1. 行動選択: a = ε-greedy(Q(s))
2. 環境との相互作用: s' = env.step(a)
3. バッファに保存: buffer.push(s, a, r, s', done)
4. ミニバッチ学習:
   a. バッファからランダムサンプリング
   b. TD目標を計算: target = r + γ * max Q_target(s', a')
   c. 損失を計算: L = MSE(Q(s,a), target)
   d. 逆伝播でQ-Networkを更新
5. 定期的にTarget Networkを更新: Q_target ← Q
6. εを徐々に減衰
```

### εの減衰

初期は ε=1.0（完全ランダム）から開始し、学習が進むにつれて ε_min まで減衰させる。

```
ε ← max(ε_min, ε * ε_decay)
```

初期は探索を重視し（環境を知るため）、学習が進むにつれて活用を増やす（学んだ知識を使うため）。

### Q-Learning → DQN の対応

| テーブルQ-Learning | DQN |
|---|---|
| Q-テーブル Q[s][a] | ニューラルネットワーク Q(s; θ) |
| テーブル更新 | 勾配降下法 |
| 各状態を独立に更新 | 汎化（似た状態で似た値を出力） |
| 小さな離散状態空間 | 大きな/連続的な状態空間 |

### 本実装の設計

DLフレームワーク（Linear, ReLU, Sequential, Adam）をそのまま使ってQ-Networkを構築している。これにより「DLとRLの接続点」を体験できる。

```python
def _build_network(state_dim, action_dim, hidden_dim):
    return Sequential([
        Linear(state_dim, hidden_dim),
        ReLU(),
        Linear(hidden_dim, hidden_dim),
        ReLU(),
        Linear(hidden_dim, action_dim),
    ])
```

## 手法の全体像

### 分類軸

```
            モデルあり           モデルなし
           ┌─────────────┬────────────────────────┐
           │             │                        │
価値ベース │ 動的計画法    │ Q-Learning, SARSA, DQN │
           │             │                        │
方策ベース │             │ REINFORCE              │
           │             │                        │
           └─────────────┴────────────────────────┘
```

### 学習の進化

```
バンディット → TD学習 → DQN
(状態なし)   (テーブル) (ニューラルネット)
     ↓
  探索と活用    →    ベルマン方程式    →    関数近似
  の基本原理        による価値推定        + 安定化技法
```

各段階が前の段階の自然な拡張になっている。バンディットで学んだ探索と活用の原理は、TD学習やDQNにそのまま引き継がれている（ε-greedy）。ベルマン方程式の考え方は動的計画法からTD学習、DQNまで一貫している。
