## 線形回帰

### 概念

線形回帰は最も基本的な教師あり学習アルゴリズムである。入力特徴量と出力の間に**線形な関係**を仮定し、その関係を表すパラメータを学習する。

「データに最もよくフィットする直線（超平面）を引く」というのが本質である。

### モデル

```
y = Xw + b
```

- `X`: 入力行列 (n_samples × n_features)
- `w`: 重みベクトル (n_features)
- `b`: バイアス（切片）
- `y`: 予測値

### 損失関数

平均二乗誤差 (MSE) を最小化する。

```
L(w, b) = (1/2n) Σ (y_i - (x_i・w + b))²
```

MSEを選ぶ理由は2つある。(1) 微分が容易で勾配計算が単純になる。(2) 誤差が正規分布に従う場合、MSE最小化は最尤推定と等価になる。

### 2つの解法

#### 勾配降下法

パラメータを損失関数の勾配の逆方向に少しずつ更新する反復的手法。

```
w ← w - α * (1/n) * X^T(Xw + b - y)
b ← b - α * (1/n) * Σ(Xw + b - y)
```

α（学習率）は更新の刻み幅を制御する。大きすぎると発散し、小さすぎると収束が遅い。

勾配降下法の本質は「目隠しで山を下る」ことに例えられる。現在地の傾斜（勾配）だけを頼りに、最も急な下り方向へ一歩ずつ進む。

#### 正規方程式

損失関数の勾配を0とおき、解析的に解を求める。

```
w = (X^T X)^{-1} X^T y
```

逆行列の計算が必要なため、特徴量数が多い場合（数万以上）は計算コストが高い。一方、学習率のチューニングが不要で、一発で最適解が得られるという利点がある。

### 設計上の判断

本実装では勾配降下法と正規方程式の両方を別クラスとして提供している。これは「同じ問題に対して異なるアプローチが存在する」ことを理解するためである。実務では scikit-learn も内部で正規方程式（特異値分解）を使っている。

## ロジスティック回帰

### 概念

名前に「回帰」とあるが、**分類**アルゴリズムである。線形回帰の出力にシグモイド関数を適用し、出力を確率 [0, 1] に変換する。

本質は「線形な決定境界で2クラスを分離する」ことである。

### モデル

```
p(y=1|x) = σ(x・w + b)
σ(z) = 1 / (1 + e^(-z))
```

シグモイド関数σは任意の実数を (0, 1) の範囲に押しつぶす。これにより出力を「クラス1に属する確率」として解釈できる。

### 損失関数（Binary Cross-Entropy）

```
L = -(1/n) Σ [y_i * log(p_i) + (1-y_i) * log(1-p_i)]
```

なぜMSEではなくCross-Entropyを使うのか？ シグモイド関数との組み合わせでMSEを使うと、損失関数が非凸になり局所最適に陥る可能性がある。Cross-Entropyは凸関数になるため、勾配降下法で大域最適解に到達できる。

### 勾配の美しさ

勾配を計算すると、線形回帰と全く同じ形になる。

```
dw = (1/n) * X^T(p - y)
db = (1/n) * Σ(p - y)
```

これはシグモイド関数とCross-Entropyの組み合わせから生まれる数学的な「奇跡」であり、指数族分布の一般的な性質に由来する。

### 線形回帰との関係

| | 線形回帰 | ロジスティック回帰 |
|---|---|---|
| タスク | 回帰 | 分類 |
| 出力 | 実数 | 確率 [0,1] |
| 活性化 | なし（恒等関数） | シグモイド |
| 損失 | MSE | Cross-Entropy |
| 勾配の形 | X^T(pred - y) | X^T(pred - y) |

## Ridge回帰・Lasso回帰

### 概念：正則化とは

正則化は**過学習を防ぐ**ための手法である。モデルの複雑さにペナルティを課すことで、訓練データに過剰にフィットすることを抑制する。

本質的な問いは「データへの適合度」と「モデルの単純さ」のトレードオフをどう取るかである。

### Ridge回帰（L2正則化）

```
L = (1/2n) * ||y - Xw||² + α * ||w||²
```

L2ペナルティ `||w||²` は重みの二乗和を小さくする。これにより全ての重みが均等に小さくなる方向に誘導される。

解析解が存在する：
```
w = (X^T X + αI)^{-1} X^T y
```

`αI` を加えることで `X^T X` が常に正則（逆行列が存在）になる。これが「Ridge（尾根）」の名前の由来で、特異な行列に「尾根」を加えて安定化させる。

**多重共線性**（特徴量間の強い相関）がある場合に特に有効。

### Lasso回帰（L1正則化）

```
L = (1/2n) * ||y - Xw||² + α * ||w||₁
```

L1ペナルティ `||w||₁` は重みの絶対値の和を小さくする。L2と異なり、**一部の重みを完全に0にする**効果がある。

これが**特徴量選択**として機能する。100個の特徴量のうち本当に重要なのが10個だけなら、Lassoは残り90個の重みを0にしてくれる。

#### 座標降下法

Lassoは L1ノルムの微分が0で不連続（絶対値関数の角）なため、通常の勾配降下法が使えない。代わりに**座標降下法**を使う。

1つの重みだけを動かし（他は固定）、最適値を求める操作を全重みに対して繰り返す。各ステップの解は**ソフト閾値関数**で与えられる：

```
w_j = soft_threshold(ρ_j, α) / z_j

soft_threshold(ρ, α) = sign(ρ) * max(|ρ| - α, 0)
```

`|ρ| < α` のとき重みが0になる。これがスパース解を生むメカニズムである。

### L1 vs L2 の幾何学的理解

制約領域の形状が鍵を握る。

- L2: 円（球）形の制約 → 等高線と円が接する点が解。円に角がないので重みが0になりにくい。
- L1: 菱形（正多面体）の制約 → 等高線と菱形が接する点が解。角があるため軸上（重み=0）で接しやすい。

## 決定木

### 概念

決定木は「はい/いいえ」の質問を連鎖させてデータを分類・予測する。人間の意思決定プロセスに最も近い機械学習アルゴリズムである。

本質は**特徴量空間を再帰的に矩形領域に分割する**ことである。

### CARTアルゴリズム

Classification And Regression Tree。各ノードで「どの特徴量のどの値で分割するか」を**貪欲に**決定する。

#### 分類：ジニ不純度

```
Gini(S) = 1 - Σ p_k²
```

`p_k` はクラスkの割合。ジニ不純度は「ランダムに選んだ2つのサンプルが異なるクラスである確率」を表す。

- 全サンプルが同じクラス → Gini = 0（純粋）
- 2クラスが半々 → Gini = 0.5（最大不純度）

分割の評価は**情報利得**で行う：

```
Gain = Gini(親) - Σ (|子|/|親|) * Gini(子)
```

全ての特徴量・全ての閾値を走査し、情報利得が最大となる分割を選ぶ。

#### 回帰：分散の削減

回帰ではジニ不純度の代わりに分散を使う。分散の削減量が最大となる分割を選ぶ。

```
Variance Reduction = Var(親) - Σ (|子|/|親|) * Var(子)
```

### なぜ貪欲法か

最適な決定木を見つける問題はNP完全であり、全ての可能な木を探索することは計算上不可能。そのため各ノードでの局所最適を選ぶ貪欲法を使う。

### 過学習と剪定

決定木は制限なく成長させると訓練データを完全に記憶してしまう。これを防ぐため：
- `max_depth`: 木の最大深さを制限
- `min_samples_split`: 分割に必要な最小サンプル数

本実装ではこの2つのハイパーパラメータで制御している。

## サポートベクターマシン

### 概念

SVMは2クラスを分離する**マージン最大の超平面**を見つけるアルゴリズムである。

マージンとは決定境界と最も近いデータ点（サポートベクター）との距離のこと。マージンを最大化することで、未知データに対する汎化性能を高める。

### 定式化

```
min (1/2)||w||² + C Σ max(0, 1 - y_i(w・x_i + b))
```

第1項はマージンの最大化（||w||が小さいほどマージンが大きい）、第2項は誤分類へのペナルティ（ヒンジ損失）。Cは両者のバランスを制御する。

### ヒンジ損失

```
hinge(y, f(x)) = max(0, 1 - y * f(x))
```

- `y * f(x) >= 1`: 正しく分類、かつマージン内にない → 損失0
- `y * f(x) < 1`: マージン内にある、または誤分類 → ペナルティ

ヒンジ損失の本質は「正しく分類していても、マージンの内側にいれば罰する」こと。これがマージン最大化を実現する。

### 実装上の工夫

本実装ではSGD（確率的勾配降下法）でヒンジ損失を直接最適化している。各サンプルについて：

- マージン内にない場合：正則化項の勾配のみ更新
- マージン内にある場合：正則化項 + ヒンジ損失の勾配で更新

カーネル法（非線形SVM）は実装していない。線形SVMの本質を理解することが目的である。

## k近傍法

### 概念

KNNは**怠惰学習**（lazy learning）の代表例。学習フェーズではデータを保存するだけで、予測時にはじめて計算を行う。

本質は「類は友を呼ぶ」。未知のデータ点の予測は、最も近いk個のデータ点の多数決（分類）または平均（回帰）で決まる。

### アルゴリズム

1. 新しいデータ点と全訓練データの距離を計算
2. 距離が小さい順にk個を選択
3. 分類：k個の多数決、回帰：k個の平均値

### kの選び方

- k=1: 最も近い1点で決定。ノイズに敏感（過学習）
- k=n: 全データの多数クラス。常に同じ予測（過少学習）
- 一般的には奇数を選び（2クラスの場合の同票を防ぐ）、交差検証で最適なkを探す

### 計算量の問題

予測時に全データとの距離計算が必要なため、データ数nに対してO(n・d)の計算量がかかる。大規模データには KD木 や Ball木 による高速化が必要になるが、本実装ではアルゴリズムの本質理解のため素朴な実装としている。

### 次元の呪い

高次元空間ではデータ点間の距離が均一化し、「近い」と「遠い」の区別がつかなくなる。これが次元の呪い (curse of dimensionality) であり、KNNが高次元データに弱い根本的な理由である。

## ナイーブベイズ

### 概念

ベイズの定理を用いた分類器。「ナイーブ」の由来は、全ての特徴量が**条件付き独立**であるという（多くの場合非現実的だが強力な）仮定にある。

### ベイズの定理

```
P(y|x) = P(x|y) * P(y) / P(x)
```

- P(y|x): 事後確率（データxが与えられたときのクラスyの確率）
- P(x|y): 尤度（クラスyのデータがxである確率）
- P(y): 事前確率（クラスyの頻度）
- P(x): エビデンス（正規化定数、分類には不要）

### 条件付き独立の仮定

```
P(x₁, x₂, ..., x_d | y) = Π P(x_i | y)
```

各特徴量が独立であると仮定することで、尤度の計算がd個の1次元分布の積になる。これが計算を劇的に簡単にする。

### ガウシアンナイーブベイズ

各特徴量がクラスごとにガウス分布に従うと仮定：

```
P(x_i | y=c) = (1/√(2πσ²_{c,i})) * exp(-(x_i - μ_{c,i})² / (2σ²_{c,i}))
```

学習は各クラスの各特徴量の平均と分散を計算するだけ。極めて高速。

### 対数空間での計算

確率の積は非常に小さい数になりアンダーフローを起こす。対数を取ることで積が和になり、数値的に安定する：

```
log P(y|x) = log P(y) + Σ log P(x_i | y)
```

### なぜ「ナイーブ」なのに実用的か

条件付き独立の仮定は通常成り立たない。しかし分類においては事後確率の「順序」だけが重要であり、確率の「値」が正確である必要はない。仮定が崩れても順序関係が保たれることが多いため、実用上は驚くほどよく機能する。

## K-Means

### 概念

K-Meansは最も広く使われる教師なし学習アルゴリズムで、データをK個のクラスタに分割する。

本質は**各データ点と所属クラスタの重心との距離の総和を最小化する**ことである。

### アルゴリズム（ロイドのアルゴリズム）

1. K個のセントロイド（重心）をランダムに初期化
2. **割り当てステップ**: 各データ点を最も近いセントロイドに割り当て
3. **更新ステップ**: 各クラスタの平均をセントロイドとして更新
4. 収束するまで 2-3 を繰り返す

### 最適化の観点

K-Meansは以下の目的関数を最小化するEM（期待値最大化）アルゴリズムの一種と見なせる：

```
J = Σ_k Σ_{x∈C_k} ||x - μ_k||²
```

- Eステップ（割り当て）: μを固定してクラスタ割り当てを最適化
- Mステップ（更新）: クラスタ割り当てを固定してμを最適化

各ステップでJは単調に減少し、有限回で収束する。ただし**局所最適解**に陥る可能性がある。

### 初期化の重要性

ランダム初期化により結果が大きく変わる。実務では K-Means++ （最初のセントロイドから遠い点を優先的に選ぶ）が使われる。本実装ではランダム初期化としている。

### K-Meansの限界

- クラスタ数Kを事前に指定する必要がある
- 球状のクラスタしか検出できない（非凸形状は苦手）
- 外れ値に敏感（平均を使うため）

## DBSCAN

### 概念

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) は**密度ベース**のクラスタリング手法。K-Meansと異なり、クラスタ数を事前に指定する必要がなく、任意の形状のクラスタを検出できる。

本質は「密度が高い領域をクラスタとし、密度が低い領域をクラスタの境界とする」ことである。

### 3種類の点

- **コア点**: eps半径内にmin_samples個以上の点がある
- **ボーダー点**: コア点ではないが、コア点のeps近傍内にある
- **ノイズ点**: どちらでもない

### アルゴリズム

1. 各点について eps 半径内の近傍点を計算
2. コア点を見つける
3. コア点から**密度到達可能**な点をすべて同じクラスタに割り当て
4. どのクラスタにも属さない点をノイズ（-1）とする

### K-Meansとの比較

| | K-Means | DBSCAN |
|---|---|---|
| クラスタ数 | 事前指定 | 自動検出 |
| クラスタ形状 | 球状 | 任意 |
| ノイズ耐性 | 低い | 高い（ノイズ点を除外） |
| パラメータ | K | eps, min_samples |
| 計算量 | O(n・K・I) | O(n²)（素朴な実装） |

## 主成分分析

### 概念

PCA (Principal Component Analysis) はデータの**分散が最大となる方向**を見つけ、その方向に射影することで次元削減を行う。

本質は「情報の損失を最小限に抑えながら、データの次元を下げる」ことである。

### アルゴリズム

1. データを中心化（平均を引く）
2. 共分散行列を計算: `C = (1/n) X^T X`
3. 共分散行列の固有値分解
4. 固有値が大きい順にk個の固有ベクトル（主成分）を選択
5. 主成分で射影: `Z = X @ W`

### なぜ共分散行列の固有ベクトルか

データの分散が最大となる方向 `w` を求める問題は：

```
max w^T C w   subject to ||w|| = 1
```

ラグランジュ乗数法で解くと `Cw = λw` となる。つまり共分散行列の固有ベクトルが分散最大の方向であり、対応する固有値がその方向の分散量を表す。

### 寄与率

各主成分がデータの分散の何%を説明するかを示す：

```
寄与率_k = λ_k / Σ λ_i
```

累積寄与率が90%に達する主成分数を選ぶのが一般的な基準。

### 逆変換

PCAは線形変換なので、低次元から元の空間に戻すことができる（ただし情報は失われている）：

```
X_reconstructed = Z @ W^T + mean
```

## t-SNE

### 概念

t-SNE (t-distributed Stochastic Neighbor Embedding) は高次元データの**可視化**に特化した非線形次元削減手法。PCAが「全体構造」を保つのに対し、t-SNEは「局所構造」（近くの点同士の関係）を保つ。

### 核心的なアイデア

1. **高次元**での類似度：ガウス分布で定義

```
p_{j|i} = exp(-||x_i - x_j||² / 2σ_i²) / Σ_{k≠i} exp(-||x_i - x_k||² / 2σ_i²)
```

2. **低次元**での類似度：Student-t分布（自由度1）で定義

```
q_{ij} = (1 + ||y_i - y_j||²)^{-1} / Σ_{k≠l} (1 + ||y_k - y_l||²)^{-1}
```

3. 2つの分布のKLダイバージェンスを最小化

### なぜStudent-t分布か（Crowding問題）

高次元空間では「中程度の距離」の点が多数存在する。これを低次元に埋め込むと、これらの点が近くに押し込められる（crowding問題）。

Student-t分布はガウス分布より**裾が重い**。つまり低次元で遠い点に対してもゼロでない確率を与えるため、crowding問題を緩和できる。これがvan der Maatenの「t-SNE」の核心的な貢献である。

### Perplexity

σ_iは各点ごとに異なり、perplexityパラメータで制御する。Perplexityは「有効な近傍数」の目安であり、二分探索で適切なσ_iを決定する。

- Perplexity小: 局所構造を重視（小さなクラスタが見える）
- Perplexity大: 大域構造を重視（大きな構造が見える）
- 一般的には5〜50の範囲

### Early Exaggeration

学習の初期段階でp_ijを数倍に増幅する。これにより低次元空間でクラスタが早期に形成され、その後の最適化で位置が微調整される。

### PCAとの使い分け

- PCA: 線形、高速、解釈可能、前処理やデータ圧縮に適する
- t-SNE: 非線形、低速、可視化専用。逆変換が定義できず、新しいデータ点の埋め込みには再計算が必要

## ランダムフォレスト

### 概念

ランダムフォレストは**バギング + 特徴量のランダム選択**によるアンサンブル学習。多数の決定木を独立に学習し、その予測を集約する。

「三人寄れば文殊の知恵」の統計的な実現である。

### なぜアンサンブルが有効か

単一の決定木は高分散（データが少し変わると予測が大きく変わる）。多数の木の予測を平均化すると分散が減少する：

```
Var(平均) = Var(個別) / n    （独立な場合）
```

ただし同じデータで学習した木は独立でない。そこで2つのランダム化を導入する。

### 2つのランダム化

1. **ブートストラップサンプリング**: 訓練データから復元抽出でサブセットを作成。各木が異なるデータで学習する。
2. **特徴量のランダム選択**: 各ノードの分割で全特徴量ではなく√d個（分類の場合）をランダムに選ぶ。木同士の相関を下げる。

### バイアス-分散トレードオフ

- 個々の木: 低バイアス・高分散
- ランダムフォレスト: 低バイアス・低分散（分散のみ削減）

木の数を増やしても過学習しない（稀な性質）。計算資源が許す限り木を増やすのが基本戦略。

## 勾配ブースティング

### 概念

勾配ブースティングは弱学習器（浅い決定木）を**逐次的に**追加し、前のモデルの**残差**を学習する手法。

ランダムフォレストが「多数の木を独立に学習」するのに対し、勾配ブースティングは「前の木の失敗を次の木が補う」。

### アルゴリズム

```
F_0(x) = mean(y)                        # 初期予測は平均値
F_m(x) = F_{m-1}(x) + η * h_m(x)       # m番目の木を加算

h_m は残差 r_m = y - F_{m-1}(x) を学習した決定木
η は学習率（shrinkage）
```

### なぜ「勾配」ブースティングか

残差 `y - F(x)` は実はMSE損失関数の**負の勾配**に等しい：

```
-∂L/∂F(x) = -(F(x) - y) = y - F(x) = 残差
```

つまり各ステップで損失関数の勾配方向に関数を更新しており、これは**関数空間での勾配降下法**と解釈できる。

MSE以外の損失関数（例：Huber損失）を使えば、外れ値にロバストなブースティングが得られる。

### 学習率の役割

学習率ηは各木の寄与を縮小する。直感に反するが、ηを小さくして木の数を増やす方が汎化性能が高い。これは「小さなステップで慎重に最適化する」ことに対応する。

### ランダムフォレストとの比較

| | ランダムフォレスト | 勾配ブースティング |
|---|---|---|
| 学習方式 | 並列（独立） | 逐次（依存） |
| 低減するもの | 分散 | バイアス（+分散） |
| 過学習リスク | 低い | あり（木数・学習率に依存） |
| チューニング | 容易 | やや難しい |
