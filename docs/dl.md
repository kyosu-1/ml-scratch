## ニューラルネットワークの基礎

### 計算グラフとしてのニューラルネット

ニューラルネットワークは**計算グラフ**である。入力から出力への計算を有向グラフとして表現し、各ノードが単純な演算（行列積、活性化関数の適用など）を行う。

学習の本質は、この計算グラフを「逆方向に」辿って各パラメータが損失にどれだけ寄与しているかを計算し（誤差逆伝播）、その情報に基づいてパラメータを更新すること。

### 本実装の設計

各層（Layer）は統一的なインターフェースを持つ：

```python
class Layer:
    def forward(self, x):   # 順伝播：入力を受け取り出力を返す
    def backward(self, grad): # 逆伝播：出力側の勾配を受け取り入力側の勾配を返す
    def params_and_grads(self): # パラメータと勾配のリストを返す
```

この設計により、層を任意に組み合わせてSequentialモデルを構築できる。PyTorchの `nn.Module` もこの考え方の延長にある。

## 全結合層

### 計算

```
y = Xw + b
```

全ての入力ニューロンが全ての出力ニューロンに接続される。最も基本的な層。

### 逆伝播の導出

出力側からの勾配を `δ = dL/dy` とすると：

```
dL/dw = X^T @ δ        (重みの勾配)
dL/db = Σ δ             (バイアスの勾配)
dL/dX = δ @ w^T         (入力への勾配、前の層に渡す)
```

行列の微分において、`y = Xw` の場合：
- wで微分 → X^T が係数
- Xで微分 → w^T が係数

これは線形代数の転置の性質から自然に導かれる。

### 重みの初期化：He初期化

```
w ~ N(0, √(2/fan_in))
```

ReLUと組み合わせる場合に分散が層を経ても維持されるよう設計されている。fan_inは入力ユニット数。

初期化が重要な理由：
- 小さすぎ → 勾配消失（深い層ほど勾配が指数的に減衰）
- 大きすぎ → 勾配爆発（深い層ほど勾配が指数的に増大）

He初期化はReLUの「半分のニューロンが0を出力する」性質を考慮して分散を2倍にしている（Xavierの2倍）。

## 活性化関数

活性化関数がない場合、多層のニューラルネットワークは1層の線形変換と等価になる（行列の積は行列）。非線形性を導入することで、任意の複雑な関数を近似できるようになる（万能近似定理）。

### ReLU

```
f(x) = max(0, x)
f'(x) = 1 (x > 0), 0 (x ≤ 0)
```

現代の深層学習で最も広く使われる活性化関数。利点：
- 勾配が0か1。シグモイドやtanhのように飽和しないため、勾配消失が起きにくい
- 計算が極めて単純（比較と選択のみ）
- スパース活性化（多くのニューロンが0を出力）

欠点：
- **Dying ReLU問題**：入力が常に負になるニューロンは永久に0を出力し、学習されなくなる

### Sigmoid

```
f(x) = 1 / (1 + e^{-x})
f'(x) = f(x)(1 - f(x))
```

出力が (0, 1) に収まるため確率の解釈が可能。ただし：
- 飽和領域で勾配がほぼ0 → 勾配消失
- 出力が0中心でない → 勾配の符号が偏る

歴史的に重要だが、隠れ層では現在ほとんど使われない。出力層での確率出力には今も使われる。

### Tanh

```
f(x) = (e^x - e^{-x}) / (e^x + e^{-x})
f'(x) = 1 - f(x)²
```

Sigmoidを [-1, 1] にスケーリングしたもの。0中心であることがSigmoidに対する利点。RNNの隠れ状態の活性化に使われることが多い。

### Softmax

```
f(x_i) = e^{x_i} / Σ e^{x_j}
```

ベクトル全体を確率分布に変換する。出力の総和が1になる。多クラス分類の出力層で使用。

数値安定性のため、最大値を引いてから計算する：
```
f(x_i) = e^{x_i - max(x)} / Σ e^{x_j - max(x)}
```

これは分子分母に同じ定数 `e^{-max(x)}` を掛けるだけなので結果は変わらない。

## 損失関数

### 平均二乗誤差 (MSE)

```
L = (1/n) Σ (y_i - ŷ_i)²
勾配: dL/dŷ = 2(ŷ - y) / n
```

回帰タスクで使用。誤差が正規分布に従う仮定のもとでの最尤推定と等価。

### 交差エントロピー (Cross-Entropy)

```
L = -(1/n) Σ Σ y_{i,k} log(p_{i,k})
```

分類タスクで使用。KLダイバージェンスの一部であり、予測確率分布と真の分布の「距離」を測る。

Softmax + Cross-Entropyの組み合わせでは、勾配が非常にシンプルになる：

```
dL/d(logits) = (softmax(logits) - y_onehot) / n
```

これが実装上の大きな利点。本実装ではCrossEntropyLossがSoftmax出力を受け取り、この簡潔な勾配を返す設計になっている。

### 二値交差エントロピー (Binary Cross-Entropy)

```
L = -(1/n) Σ [y_i log(p_i) + (1-y_i) log(1-p_i)]
```

2クラス分類用。Sigmoid出力と組み合わせる。

## 誤差逆伝播法

### 連鎖律（チェインルール）

誤差逆伝播法の数学的基礎は微分の連鎖律である：

```
dL/dx = dL/dy * dy/dx
```

計算グラフの各ノードが局所的な微分 `dy/dx` を計算し、後ろから伝播してきた勾配 `dL/dy` と掛け合わせることで `dL/dx` を得る。

### 具体例：2層ネットワーク

```
z₁ = X @ W₁ + b₁
a₁ = ReLU(z₁)
z₂ = a₁ @ W₂ + b₂
ŷ  = Softmax(z₂)
L  = CrossEntropy(ŷ, y)
```

逆伝播：
```
δ₂ = ŷ - y                    # Softmax + CE の勾配
dW₂ = a₁^T @ δ₂
δ₁ = δ₂ @ W₂^T * (z₁ > 0)    # ReLUの勾配
dW₁ = X^T @ δ₁
```

本実装では各層の `backward` メソッドがこの計算を自動的に行う。Sequentialモデルは層を逆順に辿って `backward` を連鎖させるだけである。

### 計算効率

順伝播と逆伝播は同じ計算量（定数倍の差）。これは素朴な数値微分（各パラメータを微小に変化させて損失を再計算）に比べて劇的に効率的。パラメータ数がdの場合：
- 数値微分: O(d) 回の順伝播が必要
- 逆伝播: 1回の逆伝播で全パラメータの勾配を得られる

## オプティマイザ

勾配が計算できたら、それを使ってパラメータを更新する。「どの方向にどれだけ動くか」を決めるのがオプティマイザの役割。

### SGD + Momentum

```
v_t = μ * v_{t-1} + η * g_t
w_t = w_{t-1} - v_t
```

Momentumは「慣性」を加える。過去の勾配方向への速度を維持するため：
- 谷間を素早く抜ける（一貫した方向の勾配が蓄積される）
- 振動を抑える（反対方向の勾配が相殺される）

ボールが谷を転がり落ちるアナロジー。

### Adam

```
m_t = β₁ m_{t-1} + (1-β₁) g_t        # 1次モーメント（勾配の移動平均）
v_t = β₂ v_{t-1} + (1-β₂) g_t²       # 2次モーメント（勾配二乗の移動平均）
m̂_t = m_t / (1 - β₁^t)               # バイアス補正
v̂_t = v_t / (1 - β₂^t)               # バイアス補正
w_t = w_{t-1} - η * m̂_t / (√v̂_t + ε)
```

Adamは「Adaptive Moment Estimation」の略。核心的なアイデアは：

1. **1次モーメント (m)**: SGD + Momentumと同じ効果
2. **2次モーメント (v)**: パラメータごとの学習率の自動調整。勾配が大きいパラメータは学習率を下げ、小さいパラメータは上げる
3. **バイアス補正**: 学習初期にm, vが0で初期化されるため、真の期待値より小さくなるバイアスを補正

実用上、Adamはほとんどの場合でうまく動作し、ハイパーパラメータのチューニングが最も少ない。

### 本実装の設計

オプティマイザは `step(layers)` で全層のパラメータを一括更新する。各層は `params_and_grads()` でパラメータとその勾配のリストを返す。

```python
optimizer.step(model.layers)
```

この設計によりオプティマイザと層が疎結合になり、任意のオプティマイザと層の組み合わせが可能。

## バッチ正規化

### 問題：内部共変量シフト

深いネットワークでは、前の層のパラメータ更新により各層の入力分布が変化する（内部共変量シフト）。これが学習を不安定にする。

### アイデア

各ミニバッチ内で入力を正規化し、平均0・分散1にする。

```
μ = mean(x)
σ² = var(x)
x̂ = (x - μ) / √(σ² + ε)
y = γ * x̂ + β
```

### γとβの役割

正規化だけでは表現力が失われる（全ての層の出力が平均0・分散1に制限される）。学習可能なパラメータ γ（スケール）と β（シフト）を導入することで、ネットワークが正規化を「取り消す」ことも可能にする。

恒等変換を学習できる = 正規化が不要なら元に戻せる = 少なくとも正規化しない場合より悪くならない。

### 学習時と推論時の違い

- 学習時: ミニバッチの統計量を使用し、移動平均を更新
- 推論時: 学習中に蓄積した移動平均を使用

推論時にはバッチの概念がないため、学習中の統計量を使う必要がある。

### 逆伝播

BatchNormの逆伝播は複雑に見えるが、本質的には「正規化の逆操作を勾配に適用する」こと：

```
dx = (1/N) * (1/√(σ²+ε)) * (N*dx̂ - Σdx̂ - x̂*Σ(dx̂*x̂))
```

## ドロップアウト

### 概念

学習時にランダムにニューロンを無効化（出力を0にする）することで過学習を防ぐ正則化手法。

直感的には「どのニューロンが落とされても機能するネットワーク」を学習する。これにより特定のニューロンへの過度な依存を防ぐ。

### Inverted Dropout

```
学習時: mask ~ Bernoulli(1-p),  y = x * mask / (1-p)
推論時: y = x
```

`1/(1-p)` でスケーリングする理由：学習時に平均的に `(1-p)` の割合のニューロンしか活性化しないため、推論時（全ニューロン活性化）との出力の期待値を合わせる。

### アンサンブル学習としての解釈

ドロップアウトは指数的に多くのサブネットワーク（2^n個）のアンサンブル平均を近似している。各ミニバッチで異なるサブネットワークが学習され、推論時にはそれらの平均が得られる。

## 畳み込みニューラルネットワーク

### 概念

画像などの空間的構造を持つデータに適した層。3つの重要な性質を持つ：

1. **局所受容野**: 全ピクセルではなく、小さなパッチ（カーネル）のみを見る
2. **重み共有**: 同じカーネルを画像の全位置で使い回す → パラメータ数を大幅に削減
3. **平行移動不変性**: 物体が画像内のどこにあっても同じ特徴を検出

### 畳み込み演算

カーネル（フィルタ）を入力上でスライドさせ、要素積の和を計算：

```
出力[i,j] = Σ_m Σ_n 入力[i+m, j+n] * カーネル[m, n]
```

出力サイズ：
```
H_out = (H + 2*padding - kernel_size) / stride + 1
```

### im2col による高速化

素朴なループ実装は遅い。im2colは畳み込みを行列積に変換する手法：

1. 入力の各パッチ（カーネルサイズ × カーネルサイズ）を列ベクトルとして並べる
2. カーネルも1次元に展開
3. 行列積を計算 → これが畳み込みの結果

```
im2col(入力) → (C*K*K, H_out*W_out) の行列
カーネル     → (out_channels, C*K*K) の行列
出力 = カーネル @ im2col(入力)
```

NumPyの最適化された行列積が利用でき、ループ実装に比べて桁違いに高速。

### 逆伝播

im2colの逆操作 (col2im) で入力に対する勾配を復元する。カーネルの勾配は im2col された入力と出力の勾配の行列積で計算される。

## プーリング層

### MaxPool2D

各ウィンドウ（通常2×2）内の最大値を取る。効果：

1. **ダウンサンプリング**: 特徴マップのサイズを縮小（計算量削減）
2. **平行移動に対するロバスト性**: 物体が少しずれても最大値は変わりにくい
3. **顕著な特徴の抽出**: 最大値 = 最も強く反応した特徴

逆伝播では、最大値の位置にのみ勾配を伝播する（他の位置の勾配は0）。

### Flatten

多次元テンソル (batch, C, H, W) を (batch, C×H×W) に展開する。畳み込み層の出力を全結合層に渡すために必要。逆伝播では元の形状に戻すだけ（reshape）。

## リカレントニューラルネットワーク

### 概念

系列データ（テキスト、時系列）を処理するためのアーキテクチャ。**隠れ状態**を通じて過去の情報を保持する。

### Vanilla RNN

```
h_t = tanh(W_xh @ x_t + W_hh @ h_{t-1} + b)
```

各時刻tで入力 `x_t` と前の隠れ状態 `h_{t-1}` を組み合わせて新しい隠れ状態 `h_t` を計算する。隠れ状態が「記憶」として機能する。

### BPTT（通時的誤差逆伝播法）

RNNの逆伝播は時間を遡って行う。時刻Tから時刻1まで、各時刻での勾配を計算して蓄積する：

```
for t in reversed(range(T)):
    dtanh = dh * (1 - h_t²)         # tanhの勾配
    dW_xh += x_t^T @ dtanh
    dW_hh += h_{t-1}^T @ dtanh
    dh_next = dtanh @ W_hh^T         # 前の時刻への勾配
```

### 勾配消失・爆発問題

長い系列では勾配が `W_hh` の繰り返し乗算により指数的に減衰（消失）または増大（爆発）する：

```
dL/dh_1 = dL/dh_T * Π_{t=2}^{T} (diag(1-h_t²) @ W_hh)
```

`W_hh` の最大特異値が 1 未満なら消失、1 より大きければ爆発。これがVanilla RNNの根本的な問題であり、LSTMとGRUが生まれた動機。

## LSTM

### 概念

Long Short-Term Memory。勾配消失問題を解決するために設計されたRNNの拡張。核心は**セル状態**（cell state）と**ゲート機構**。

### 3つのゲート

```
f_t = σ(W_f @ [h_{t-1}, x_t] + b_f)     # 忘却ゲート
i_t = σ(W_i @ [h_{t-1}, x_t] + b_i)     # 入力ゲート
o_t = σ(W_o @ [h_{t-1}, x_t] + b_o)     # 出力ゲート

c̃_t = tanh(W_c @ [h_{t-1}, x_t] + b_c)  # 候補セル値

c_t = f_t * c_{t-1} + i_t * c̃_t          # セル状態の更新
h_t = o_t * tanh(c_t)                     # 隠れ状態の出力
```

#### 忘却ゲート (f_t)

「前の記憶のどの部分を忘れるか」を制御する。出力は [0,1] で、0なら完全に忘却、1なら完全に保持。

#### 入力ゲート (i_t)

「新しい情報のどの部分を記憶するか」を制御する。

#### 出力ゲート (o_t)

「記憶のどの部分を出力するか」を制御する。

### なぜ勾配消失が解決されるか

セル状態の更新式に注目：

```
c_t = f_t * c_{t-1} + i_t * c̃_t
```

`c_t` から `c_{t-1}` への勾配は `f_t`（忘却ゲート）そのもの。忘却ゲートが1に近ければ、勾配はほぼ減衰せずに伝播する。これが「情報の高速道路（highway）」として機能し、長距離の依存関係を学習できる。

### 実装の工夫

4つのゲートの重み行列を1つに結合して計算効率を高めている：

```python
# 4つのゲートを一括計算
gates = concat @ self.W + self.b  # (batch, 4*hidden_size)
f, i, c_tilde, o = gates を4分割
```

## GRU

### 概念

Gated Recurrent Unit。LSTMの簡略版で、パラメータ数が少なく計算が速い。セル状態を廃止し、ゲートを2つに削減。

### 2つのゲート

```
z_t = σ(W_z @ [h_{t-1}, x_t])     # 更新ゲート
r_t = σ(W_r @ [h_{t-1}, x_t])     # リセットゲート

h̃_t = tanh(W_h @ [r_t * h_{t-1}, x_t])  # 候補隠れ状態
h_t = (1 - z_t) * h_{t-1} + z_t * h̃_t    # 隠れ状態の更新
```

#### 更新ゲート (z_t)

LSTMの忘却ゲートと入力ゲートを統合したもの。`z_t` が大きいほど新しい情報を取り入れ、小さいほど前の状態を保持する。

`1 - z_t` と `z_t` の和が常に1であることがポイント。忘却と入力が連動する。

#### リセットゲート (r_t)

過去の隠れ状態をどの程度「リセット」するかを制御。r_t=0 なら過去を完全に無視して現在の入力のみから候補状態を計算する。

### LSTM vs GRU

- パラメータ数: GRU < LSTM（ゲートが2つ vs 3つ + セル状態）
- 性能: タスクによってどちらが良いかは変わる
- 計算速度: GRUの方が速い
- 選択基準: データが少ない場合はGRU、十分なデータがある場合はLSTM

## Attention機構

### 問題：固定長ボトルネック

従来のseq2seqモデルでは、エンコーダの出力を固定長ベクトルに圧縮する必要があった。長い入力系列では情報が失われる。

### 核心的なアイデア

デコーダが各時刻で入力系列の**どの部分に注目するか**を動的に決定する。

### Scaled Dot-Product Attention

```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```

- **Q (Query)**: 「何を探しているか」
- **K (Key)**: 「何を持っているか」
- **V (Value)**: 「実際の情報」

QとKの内積で「関連度」を計算し、softmaxで正規化して注意重みとし、Vの加重平均を取る。

#### なぜ √d_k でスケーリングするか

d_kが大きいと内積の値が大きくなり、softmaxの出力が極端に偏る（ほぼone-hotになる）。スケーリングにより内積の分散を1に正規化し、softmaxが適切に機能するようにする。

QとKの各要素が平均0・分散1の独立な確率変数なら、内積の分散はd_kになる。√d_kで割ることで分散を1に戻す。

### 自己注意（Self-Attention）

Q, K, Vがすべて同じ入力から生成される場合。系列内の各位置が他の全位置との関係を計算する。これにより系列の長距離依存関係を直接捉えることができる。

RNNが系列を順次処理するのに対し、Self-Attentionは全位置間の関係を一度に計算する。計算量はO(n²)だが並列化可能。

## Transformer

### 概念

"Attention Is All You Need" (Vaswani et al., 2017)。RNNを完全に排除し、Attention機構のみでseq2seqモデルを構築。現代のLLMの基盤アーキテクチャ。

### Multi-Head Attention

単一のAttentionでは1つの「関係性」しか捉えられない。Multiple headsで異なる部分空間（異なる種類の関係性）を並列に学習する。

```
head_i = Attention(Q @ W_q^i, K @ W_k^i, V @ W_v^i)
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) @ W_o
```

例えば8ヘッドのAttentionでは、あるヘッドは「構文的な関係」、別のヘッドは「意味的な関係」を学習するかもしれない。

### Transformer Encoderブロック

```
x → Multi-Head Attention → Add & LayerNorm → FFN → Add & LayerNorm → 出力
        ↑_____残差接続_____↑                  ↑___残差接続___↑
```

#### 残差接続

```
output = LayerNorm(x + SubLayer(x))
```

入力をそのままショートカットして加算する。効果：
- 勾配が直接伝播するパスを確保（勾配消失を防ぐ）
- 各サブレイヤーは「残差」（元の入力との差分）のみ学習すればよい

#### Layer Normalization

Batch Normalizationが「バッチ方向」で正規化するのに対し、Layer Normalizationは「特徴量方向」で正規化する。

```
LN(x) = γ * (x - μ) / √(σ² + ε) + β
μ, σ²は各サンプルの特徴量方向の統計量
```

系列長が可変のタスクでバッチ方向の正規化は不適切なため、Layer Normを使用。

#### FFN (Feed-Forward Network)

```
FFN(x) = ReLU(x @ W₁ + b₁) @ W₂ + b₂
```

位置ごとに独立に適用される2層のニューラルネットワーク。中間層の次元は通常 4 × d_model。

Attentionが「位置間の関係」を捉えるのに対し、FFNは「各位置の特徴量を変換する」役割。

## 埋め込み層

### 概念

離散的なトークンID（単語、文字など）を連続的な密ベクトルに変換する。

### なぜ必要か

- One-hot表現は次元が語彙数に等しく（数万〜数十万）、非常にスパース
- 意味的な類似度を表現できない（全ての単語ペアが等距離）

埋め込みは低次元の密ベクトルで、意味的に近い単語が近い位置に配置される。

### 実装

実質的にはルックアップテーブル（重み行列のインデキシング）：

```python
def forward(self, indices):
    return self.weights[indices]  # 該当行を取り出すだけ
```

逆伝播では、該当インデックスの行に勾配を加算する：

```python
def backward(self, grad):
    np.add.at(self.grad_weights, self.indices, grad)
```

`np.add.at` を使う理由：同じインデックスが複数回出現する場合に勾配を正しく累積するため（通常の `+=` では最後の値で上書きされる）。
